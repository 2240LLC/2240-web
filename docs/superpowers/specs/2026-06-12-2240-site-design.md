# 2240.us ‚Äî Site Design Spec
*2026-06-12*

---

## Overview

2240.us is the web presence for the 2240 umbrella brand covering two artist projects: **Toby Brown** and **Tonsure**. The site is built in plain HTML/CSS/JS ‚Äî no framework, no build tools ‚Äî and deployed to Cloudflare Pages from a Git repository.

The site has three core ideas working together:

1. **Chameleon catalogue**: every release has its own visual personality (palette, grain, glow) that floods the page when focused
2. **Waveform player**: the landing page centers on a scrolling waveform audio player for the latest release
3. **Persistent sampler**: a draggable, resizable floating window ‚Äî styled like a Teenage Engineering Pocket Operator ‚Äî that visitors can open on any page and use to sample audio from any release

## Pages

### Landing page (`index.html`)

**Navigation** ‚Äî topbar, every page:
```
2240          Home  Releases  Contact
```
`2240` wordmark links to `/`. Active page link has bottom border underline.

**Layout** ‚Äî full-viewport, flex column:
1. Topbar
2. Release metadata (eyebrow, title, artist)
3. Waveform player
4. Transport controls
5. Footer

**Palette**: landing page defaults to the latest release's `data-mode` ‚Äî currently `tonsure`. When a new release launches, this value changes.

**Waveform player**:
- Full-width `<canvas>` element, ~160px tall
- Bars: 3px wide, 1.5px gap, ~420 bars across full width
- Waveform data: decoded from the self-hosted MP3 via `AudioContext.decodeAudioData()` on page load, downsampled to bar count
- Playback via `AudioBufferSourceNode` ‚Äî re-created on each play (standard Web Audio pattern)
- Scrolling style: **fixed center playhead**, bars scroll left as audio plays
- Played bars: `rgba(239,239,239, 0.55 + amplitude √ó 0.35)` (brighter)
- Unplayed bars: `rgba(239,239,239, 0.1 + amplitude √ó 0.1)` (ghosted)
- Edge fades: `linear-gradient` overlays on left/right to soften the ends
- Click-to-scrub: clicking the canvas seeks to that position
- Animated with `requestAnimationFrame`

**Transport row** (below waveform):
- Play/pause circle button (36px, 1px border, triangle/pause icon)
- Time display: `current / total` in tabular-nums monospace
- Streaming links flush right: Spotify ¬∑ Apple Music ¬∑ Bandcamp

**Footer**: `¬© 2026 2240` left, `All releases ‚Üí` right linking to `releases.html`

## Persistent Sampler

A floating instrument window injected into every page by `js/sampler.js`. Inspired by the Teenage Engineering OP-1 and Pocket Operator ‚Äî tactile, bold, slightly toy-like but musically serious.

### Window behaviour

- **Position**: `position: fixed`, default bottom-right, `z-index: 9000`
- **Draggable**: drag by the title bar ‚Äî `mousedown` + `mousemove` + `mouseup` on `document`
- **Resizable**: 8-handle CSS resize handles at corners and edges using `mousedown` + delta tracking
- **Default size**: 480 √ó 320px. Min: 320 √ó 240px. Max: 90vw √ó 90vh.
- **Collapsed state**: click minimize ‚Üí collapses to title bar only (48px tall, full width retained). Click again to restore.
- **Trigger**: a persistent `[S]` button in the bottom-right corner of every page (fixed, `z-index: 8999`). Click to show/hide the sampler.
- **State persistence**: window position, size, collapsed state, active mode, and current source saved to `sessionStorage` on every change and restored on page load.

### Visual design (OP-1 / Pocket Operator aesthetic)

- **Chassis**: dark charcoal (`#1a1a1a`) body, `1px solid #333` border, subtle `box-shadow`
- **Title bar**: `#111`, monospace `SAMPLER` label in uppercase, drag affordance dots, minimize `‚Äî` and close `√ó` buttons
- **LCD display area**: `#0a120a` background (classic green-tinted LCD), monospace text readout showing current mode, source, playback position
- **Mode buttons**: three pill/rectangle buttons ‚Äî `MPC` ¬∑ `LOOP` ¬∑ `SLICE` ‚Äî in bold uppercase. Active mode has a colored indicator. Color = active source's `--accent` value (so sampling Tonsure = white accent, Washboard = cobalt, etc.)
- **Source selector**: small dropdown or button-strip listing all available releases. Selecting a source loads that release's audio buffer.
- **Pads / waveform**: content area changes per mode (see below)
- **Encoders**: CSS-drawn circular knobs for pitch and volume ‚Äî click + drag vertically to adjust

### Audio engine (`js/sampler.js`)

One `AudioContext` is shared across the **entire site** ‚Äî both the landing page player and the sampler use the same instance. It is created once in `sampler.js` and exposed as `window.AUDIO_CTX`; `player.js` reads it from there rather than creating its own. This avoids the browser's concurrent-context limit (~6) and keeps the door open for future audio routing between the player and sampler.

All sampler audio runs through:
```
AudioBufferSourceNode ‚Üí GainNode (volume) ‚Üí AudioContext.destination
```

**Source loading**: when a source is selected, fetch the corresponding MP3, decode via `AudioContext.decodeAudioData()`, store the `AudioBuffer`. Subsequent loads of the same source reuse the cached buffer ‚Äî no re-fetch.

Buffer cache: `Map<string, AudioBuffer>` keyed by release ID. Populated on first selection, persists for the session.

**Triggering a slice** (used by all modes):
```js
function triggerSlice(buffer, startSec, durationSec, opts = {}) {
  const src = ctx.createBufferSource();
  src.buffer = buffer;
  src.playbackRate.value = opts.pitch ?? 1.0;
  src.loop = opts.loop ?? false;
  if (src.loop) {
    src.loopStart = startSec;
    src.loopEnd   = startSec + durationSec;
  }
  const gain = ctx.createGain();
  gain.gain.value = opts.volume ?? 1.0;
  src.connect(gain).connect(ctx.destination);
  src.start(0, startSec, src.loop ? undefined : durationSec);
  return { src, gain };
}
```

### Mode: MPC (Pad Grid)

- 4√ó4 grid of 16 pads
- Each pad stores a `{ startSec, durationSec }` slice definition and a color
- **Assigning a slice**: select a region on the mini-waveform display, then click an empty pad to assign
- **Triggering**: mouse click on pad, or keyboard (rows: `1234` / `qwer` / `asdf` / `zxcv`)
- Pad lights up on trigger, briefly shows amplitude as brightness
- Pad colors follow the active source's palette

### Mode: Loop

- Full-width mini-waveform of the loaded source (canvas, ~60px tall)
- Two draggable handles define loop start and end
- Press play to loop the selected region
- **Pitch knob**: `playbackRate` 0.25‚Äì4.0 (displayed as semitones relative to original)
- **Volume knob**: 0‚Äì1.0
- Loop plays until stopped or mode changes
- Adjusting the region while playing stops the current loop and immediately restarts from the new start point

### Mode: Slice

- Same mini-waveform display
- Click anywhere to instantly trigger from that point
- Hold to loop from that point; release to stop
- No quantization ‚Äî fully gestural/exploratory
- Multiple simultaneous triggers allowed (polyphonic)

### Future audio sources

Any new audio added to the site is added to the sampler's source list by registering it in a `SOURCES` array in `sampler.js`:
```js
const SOURCES = [
  { id: 'tonsure',   label: 'You Are The Man ‚Äî Tonsure',   file: 'audio/tonsure-you-are-the-man.mp3',   mode: 'tonsure' },
  { id: 'washboard', label: 'Washboard ‚Äî Toby Brown',       file: 'audio/toby-brown-washboard.mp3',       mode: 'washboard' },
  { id: 'patient',   label: 'The Patient ‚Äî Toby Brown',     file: 'audio/toby-brown-the-patient.mp3',     mode: 'patient' },
  { id: 'statues',   label: 'Is It Even Easier? ‚Äî Toby Brown', file: 'audio/toby-brown-is-it-even-easier.mp3', mode: 'statues' },
];
```

## Audio files

Self-hosted MP3s committed to the repository under `audio/`. Deployed via Cloudflare Pages ‚Äî Cloudflare's CDN serves them from cache after the first load, so repeat playback is fast.

Format: MP3, 320kbps. A 3‚Äì4 minute track is ~7‚Äì9MB ‚Äî well under Cloudflare Pages' 25MB per-file limit. Git repo size is acceptable for the current catalogue (4 tracks ‚âà 30‚Äì35MB). If the catalogue grows significantly, migrate audio to Cloudflare R2.

## Accessibility baseline

- All interactive elements keyboard-accessible
- `aria-label` on icon-only buttons (play, pad triggers)
- Sampler trigger button has `aria-expanded` state
- Waveform canvas has `role="img"` with descriptive `aria-label`
- Streaming links open in new tab with `aria-label` including destination name

## Not in scope

- Server-side rendering or CMS
- User accounts or saved sessions
- Mobile-optimized sampler layout (sampler is desktop-first; on small viewports it defaults to a minimal collapsed state)
- MIDI input (possible future enhancement)
- Recording/exporting sampler output
